## <a id="Edit"></a>Edit

If a parameter is a rate parameter (such as ``u_max`` with unit ``/day``), its unit **must end with** ``/d``, ``d-1``, ``/day``, ``day-1``, ``1/d``, ``1/day``, ``/h``, ``h-1``, ``/hour``, ``hour-1``, ``1/h``, ``1/hour``, ``/m``, ``m-1``, ``/minute``, ``minute-1``, ``1/m``, ``1/minute``, ``/s``, ``s-1``, ``/second``, ``second-1``, ``1/s`` or ``1/second``, this will help to unify the time unit through the whole calcualtion.


## <a id="Release"></a>Release

In the Version tab you can publish different versions, each with a unique version number.
